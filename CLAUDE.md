# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## What This Is

Agent Protocols Graph — an interactive knowledge graph app (`knowledge-graph.html`) covering AI agent protocols and technologies (MCP, A2A, Skills, etc.). No build step, no dependencies, no server — open the HTML file in a browser.

## Architecture

- **`knowledge-graph.html`** — The entire app (~3200 lines). Contains HTML, CSS, and JS inline. All state stored in localStorage (7 keys: graph data, course progress, bookmarks, SRS, notes, gamification, paths). Graph data is defined in the `defaultData()` function which returns nodes and edges.
- **`crawler.mjs`** — Node.js CLI tool that monitors URLs for changes and generates timeline entries. Requires `knowledge-graph-data.json` to exist (exported from the UI or generated by `extract-data.mjs`). Caches crawl state in `.crawl-cache/`.
- **`extract-data.mjs`** — Extracts the `defaultData()` object from the HTML file into `knowledge-graph-data.json` for the crawler pipeline.

**Data flow:** The HTML app is the source of truth for graph structure. To use the crawler: extract data → crawl → re-import updated JSON into the UI.

## Commands

```bash
# Extract graph data from HTML for the crawler
node extract-data.mjs

# Crawl all monitored source URLs
node crawler.mjs

# Crawl a specific topic
node crawler.mjs --topic mcp

# Add a source URL to monitor
node crawler.mjs --add-source <topic-id> <url>

# List all monitored URLs
node crawler.mjs --list

# Auto-discover source URLs from well-known list
node crawler.mjs --discover

# Generate daily digest and post to Discord
node crawler.mjs --digest --discord <webhook-url>
```

## CI/CD

Two GitHub Actions workflows in `.github/workflows/`:

- **`daily-crawl.yml`** — Runs at 08:00 UTC daily (and on `workflow_dispatch`). Extracts graph data if `knowledge-graph-data.json` is missing, discovers sources, crawls all URLs, generates a digest, and commits changes. Posts to Discord if the `DISCORD_WEBHOOK` secret is set.
- **`pages.yml`** — Deploys `knowledge-graph.html` to GitHub Pages on pushes to `main` and after each successful daily crawl. Copies the HTML as `_site/index.html`.

## Key Conventions

- The HTML file is entirely self-contained — no external CDN dependencies, no imports. All CSS and JS are inline.
- Topic nodes have an `id` (kebab-case like `mcp`, `a2a`, `skills-over-mcp`) used as keys throughout the app and crawler.
- The crawler uses Node.js built-in `fetch` (Node 18+). No npm dependencies.
- There are no automated tests. Critical logic lives in `defaultData()`, the markdown parser, quiz scoring, and the SM-2 spaced repetition scheduler.
- Feature roadmap and specs are tracked in `docs/FEATURES.md`.
